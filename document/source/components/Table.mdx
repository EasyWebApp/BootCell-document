> ##### Conveying meaning to assistive technologies
>
> Using color to add meaning only provides a visual indication,
> which will not be conveyed to users of assistive technologies – such as screen readers.
> Ensure that information denoted by the color is either obvious from
> the content itself (e.g. the visible text), or is included through alternative means,
> such as additional text hidden with the `.sr-only` class.

Assign `sm`, `md`, `lg` or `xl` to `responsive` property,
making the table scroll horizontally at each `max-width` breakpoint of up to (but not including)
576px, 768px, 992px, and 1120px, respectively.

> Note that since browsers do not currently support [range context queries][1],
> we work around the limitations of [min- and max- prefixes][2] and viewports with fractional widths
> (which can occur under certain conditions on high-dpi devices, for instance)
> by using values with higher precision for these comparisons.

## Responsive tables

Responsive tables allow tables to be scrolled horizontally with ease.

> ##### Vertical clipping/truncation
>
> Responsive tables make use of `overflow-y: hidden`, which clips off any content that goes beyond the bottom or top edges of the table.
> In particular, this can clip off dropdown menus and other third-party widgets.

### Always responsive

`<Table />` is responsive & horizontally scrollable defaultly.

<Example>
    <Table>
        <TableRow type="head">
            <th>#</th>
            {[1, 2, 3, 4, 5, 6, 7, 8, 9].map(() => (
                <th>Heading</th>
            ))}
        </TableRow>
        {[1, 2, 3].map(index => (
            <TableRow>
                <th>{index}</th>
                {[1, 2, 3, 4, 5, 6, 7, 8, 9].map(() => (
                    <td>Cell</td>
                ))}
            </TableRow>
        ))}
    </Table>
</Example>

### Breakpoint specific

Use `responsive={'sm'|'md'|'lg'|'xl'}` as needed to create responsive tables up to a particular breakpoint.
From that breakpoint and up, the table will behave normally and not scroll horizontally.

**These tables may appear broken until their responsive styles apply at specific viewport widths.**

<Example>
    <Table responsive="sm">
        <TableRow type="head">
            <th>#</th>
