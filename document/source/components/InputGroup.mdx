## Basic example

Place one add-on or button on either side of an input. You may also place one on both sides of an input.

<Example>
    <InputGroup placeholder="Username" aria-label="Username" prepend="@" />
    <InputGroup
        placeholder="Recipient's username"
        aria-label="Recipient's username"
        append="@example.com"
    />
    <label for="basic-url">Your vanity URL</label>
    <InputGroup
        id="basic-url"
        placeholder="Username"
        aria-label="Username"
        prepend="https://example.com/users/"
    />
    <InputGroup
        type="number"
        aria-label="Amount (to the nearest dollar)"
        prepend="$"
        append=".00"
    />
    <InputGroup
        is="textarea"
        aria-label="With textarea"
        prepend="With textarea"
    />
</Example>

```TSX
import { render, createCell, Fragment } from 'web-cell';
import { InputGroup } from 'boot-cell/source/Form/InputGroup';

render(
    <Fragment>
        <InputGroup placeholder="Username" aria-label="Username" prepend="@" />
        <InputGroup
            placeholder="Recipient's username"
            aria-label="Recipient's username"
            append="@example.com"
        />
        <InputGroup
            id="basic-url"
            placeholder="Username"
            aria-label="Username"
            prepend="https://example.com/users/"
        />
        <InputGroup
            type="number"
            aria-label="Amount (to the nearest dollar)"
            prepend="$"
            append=".00"
        />
        <InputGroup
            is="textarea"
            aria-label="With textarea"
            prepend="With textarea"
        />
    </Fragment>
);
```

## Wrapping

Input groups wrap by default via `flex-wrap: wrap` in order to accommodate custom form field validation within an input group.
You may disable this with `.flex-nowrap`.

<Example>
    <InputGroup
        className="flex-nowrap"
        placeholder="Username"
        aria-label="Username"
        prepend="@"
    />
</Example>

```TSX
import { render, createCell } from 'web-cell';
import { InputGroup } from 'boot-cell/source/Form/InputGroup';

render(
    <InputGroup
        className="flex-nowrap"
        placeholder="Username"
        aria-label="Username"
        prepend="@"
    />
);
```

## Sizing

Add the relative form `size` property to the `<InputGroup />` itself
and contents within will automatically resize —
no need for repeating the form control `size` property on each element.

**Sizing on the individual input group elements isn’t supported.**

<Example>
    <InputGroup size="sm" prepend="Small" />
    <InputGroup prepend="Default" />
    <InputGroup size="lg" prepend="Large" />
</Example>

```TSX
import { render, createCell, Fragment } from 'web-cell';
import { InputGroup } from 'boot-cell/source/Form/InputGroup';

render(
    <Fragment>
        <InputGroup size="sm" prepend="Small" />
        <InputGroup prepend="Default" />
        <InputGroup size="lg" prepend="Large" />
    </Fragment>
);
```

## Checkboxes and radios

Place any checkbox or radio option within an input group’s addon instead of text.

<Example>
    <InputGroup
        prepend={
            <input
                type="checkbox"
                aria-label="Checkbox for following text input"
            />
        }
        aria-label="Text input with checkbox"
    />
    <InputGroup
        prepend={
            <input
                type="radio"
                aria-label="Radio button for following text input"
            />
        }
        aria-label="Text input with radio button"
    />
</Example>

```TSX
import { render, createCell, Fragment } from 'web-cell';
import { InputGroup } from 'boot-cell/source/Form/InputGroup';

render(
    <Fragment>
        <InputGroup
            prepend={
                <input
                    type="checkbox"
                    aria-label="Checkbox for following text input"
                />
            }
            aria-label="Text input with checkbox"
        />
        <InputGroup
            prepend={
                <input
                    type="radio"
                    aria-label="Radio button for following text input"
                />
            }
            aria-label="Text input with radio button"
        />
    </Fragment>
);
```

## Multiple inputs

While multiple `<input>`s are supported visually,
validation styles are only available for input groups with a single `<input>`.

<Example>
    <InputGroup prepend="First and last name">
        <Field aria-label="First name" />
        <Field aria-label="Last name" />
    </InputGroup>
</Example>

```TSX
import { render, createCell } from 'web-cell';
import { InputGroup } from 'boot-cell/source/Form/InputGroup';
import { Field } from 'boot-cell/source/Form/Field';

render(
    <InputGroup prepend="First and last name">
        <Field aria-label="First name" />
        <Field aria-label="Last name" />
    </InputGroup>
);
```

## Multiple addons

Multiple add-ons are supported and can be mixed with checkbox and radio input versions.

<Example>
    <InputGroup
        prepend={['$', '0.00']}
        aria-label="Dollar amount (with dot and two decimal places)"
    />
    <InputGroup
        append={['$', '0.00']}
        aria-label="Dollar amount (with dot and two decimal places)"
    />
</Example>

```TSX
import { render, createCell, Fragment } from 'web-cell';
import { InputGroup } from 'boot-cell/source/Form/InputGroup';

render(
    <Fragment>
        <InputGroup
            prepend={['$', '0.00']}
            aria-label="Dollar amount (with dot and two decimal places)"
        />
        <InputGroup
            append={['$', '0.00']}
            aria-label="Dollar amount (with dot and two decimal places)"
        />
    </Fragment>
);
```

## Custom forms

Input groups include support for custom selects and custom file inputs.
Browser default versions of these are not supported.

## Accessibility

Screen readers will have trouble with your forms if you don’t include a label for every input.
For these input groups, ensure that any additional label or functionality is conveyed to assistive technologies.

The exact technique to be used (`<label>` elements hidden using the `.sr-only` class,
or use of the `aria-label` and `aria-labelledby` attributes, possibly in combination with `aria-describedby`)
and what additional information will need to be conveyed will vary depending on the exact type of interface widget you’re implementing.
The examples in this section provide a few suggested, case-specific approaches.
