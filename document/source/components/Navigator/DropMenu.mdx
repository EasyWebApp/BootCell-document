## Overview

DropMenu are toggleable, contextual overlays for displaying lists of links and more.
They're toggled by clicking, not by hovering; this is [an intentional design decision][1].

## Accessibility

The [<abbr title="Web Accessibility Initiative">WAI</abbr> <abbr title="Accessible Rich Internet Applications">ARIA</abbr>][2]
standard defines an actual [`role="menu"` widget][3], but this is specific to application-like menus which trigger actions or functions.

<abbr title="Accessible Rich Internet Applications">ARIA</abbr> menus can only contain
menu items, checkbox menu items, radio button menu items, radio button groups, and
sub-menus.

BootCell's dropmenus, on the other hand, are designed to be generic and applicable to a variety of situations and markup structures.
For instance, it is possible to create dropmenus that contain additional inputs and form controls, such as search fields or login forms.

BootCell add built-in support for most standard keyboard menu interactions,
such as the ability to move through individual `<DropMenuItem />` components using the cursor keys and close the menu with the <kbd>ESC</kbd> key.

## Examples

### Single button

<Example>
    <DropMenu buttonColor="primary" caption="Dropdown button">
        <DropMenuItem>Action</DropMenuItem>
        <DropMenuItem>Another action</DropMenuItem>
        <DropMenuItem>Something else here</DropMenuItem>
    </DropMenu>
</Example>

```TSX
import { render, createCell } from 'web-cell';
import { DropMenu, DropMenuItem } from 'boot-cell/source/Navigator/DropMenu';

render(
    <DropMenu buttonColor="primary" caption="Dropdown button">
        <DropMenuItem>Action</DropMenuItem>
        <DropMenuItem>Another action</DropMenuItem>
        <DropMenuItem>Something else here</DropMenuItem>
    </DropMenu>
);
```

The best part is you can do this with any button variant, too:

<Example>
    {[
        'Primary',
        'Secondary',
        'Success',
        'Danger',
        'Warning',
        'Info',
        'Light',
        'Dark'
    ].map(color => (
        <DropMenu buttonColor={color.toLowerCase()} caption={color}>
            <DropMenuItem>Action</DropMenuItem>
            <DropMenuItem>Another action</DropMenuItem>
            <DropMenuItem>Something else here</DropMenuItem>
            <DropMenuItem />
            <DropMenuItem>Separated link</DropMenuItem>
        </DropMenu>
    ))}
</Example>

```TSX
import { render, createCell } from 'web-cell';
import { DropMenu, DropMenuItem } from 'boot-cell/source/Navigator/DropMenu';

render([
    'Primary',
    'Secondary',
    'Success',
    'Danger',
    'Warning',
    'Info',
    'Light',
    'Dark'
].map(color => (
    <DropMenu buttonColor={color.toLowerCase()} caption={color}>
        <DropMenuItem>Action</DropMenuItem>
        <DropMenuItem>Another action</DropMenuItem>
        <DropMenuItem>Something else here</DropMenuItem>
        <DropMenuItem />
        <DropMenuItem>Separated link</DropMenuItem>
    </DropMenu>
)));
```

## Menu items

### Active

Add `active` property to items in the dropdown to **style them as active**.

<Example>
    <DropMenu buttonColor="primary" caption="Dropdown">
        <DropMenuItem href="#">Regular link</DropMenuItem>
        <DropMenuItem href="#" active>
            Active link
        </DropMenuItem>
        <DropMenuItem href="#">Another link</DropMenuItem>
    </DropMenu>
</Example>

```TSX
import { render, createCell } from 'web-cell';

render(
```

```TSX
    </DropMenu>
);
```

## Menu alignment

By default, a drop menu is automatically positioned 100% from the top and along the left side of its parent.
Add `alignType="right"` to a `<DropMenu />` to right align the dropdown menu.

<Example>
    <DropMenu
        alignType="right"
        buttonColor="primary"
        caption="Right-aligned menu"
    >
        <DropMenuItem>Action</DropMenuItem>
        <DropMenuItem>Another action</DropMenuItem>
        <DropMenuItem>Something else here</DropMenuItem>
    </DropMenu>
</Example>

```TSX
import { render, createCell } from 'web-cell';
import { DropMenu, DropMenuItem } from 'boot-cell/source/Navigator/DropMenu';

render(
    <DropMenu
        alignType="right"
        buttonColor="primary"
        caption="Right-aligned menu"
    >
        <DropMenuItem>Action</DropMenuItem>
        <DropMenuItem>Another action</DropMenuItem>
        <DropMenuItem>Something else here</DropMenuItem>
    </DropMenu>
);
```

### Forms

Put a form within a dropdown menu, or make it into a dropdown menu,
and use [margin or padding utilities][4] to give it the negative space you require.

<Example>
    <DropMenu buttonColor="primary" caption="Dropdown form">
        <form className="px-4 py-3">
            <FormField
                type="email"
                label="Email address"
                placeholder="email@example.com"
            />
            <FormField
                type="password"
                label="Password"
                placeholder="Password"
            />
            <FormField>
                <ToggleField type="checkbox">Remember me</ToggleField>
            </FormField>
            <Button type="submit">Sign in</Button>
        </form>
        <DropMenuItem />
        <DropMenuItem href="#">New around here? Sign up</DropMenuItem>
        <DropMenuItem href="#">Forgot password?</DropMenuItem>
    </DropMenu>
</Example>

```TSX

    <DropMenu buttonColor="primary" caption="Dropdown form">
```
