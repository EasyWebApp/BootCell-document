## Example

The most basic list group is an unordered list with list items and the proper classes.
Build upon it with the options that follow, or with your own CSS as needed.

<Example>
    <ListGroup
        activeIndex={0}
        list={[
            { content: 'WebCell' },
            { content: 'Cell Router', disabled: true },
            { content: 'BootCell' },
            { content: 'KoAJAX' },
            { content: 'MarkCell' }
        ]}
    />
</Example>

```JavaScript
import { render } from 'web-cell';
import { ListGroup } from 'boot-cell/source/Content/ListGroup';

render(
    <ListGroup
        activeIndex={0}
        list={[
            { content: 'WebCell' },
            { content: 'Cell Router', disabled: true },
            { content: 'BootCell' },
            { content: 'KoAJAX' },
            { content: 'MarkCell' }
        ]}
    />
);
```

## Flush

Add `flush` property to remove some borders and rounded corners to render list group items edge-to-edge in a parent container (e.g., cards).

<Example>
    <ListGroup
        flush
        list={[
            { content: 'WebCell' },
            { content: 'Cell Router' },
            { content: 'BootCell' },
            { content: 'KoAJAX' },
            { content: 'MarkCell' }
        ]}
    />
</Example>

```JavaScript
import { render } from 'web-cell';

    <ListGroup
        flush
        list={[
            { content: 'WebCell' },
            { content: 'Cell Router' },
            { content: 'BootCell' },
            { content: 'KoAJAX' },
            { content: 'MarkCell' }
        ]}
    />
)
```

## Horizontal

Add `horizontal` property to change the layout of list group items from vertical to horizontal across all breakpoints.
Alternatively, choose a responsive variant `sm`, `md`, `lg` & `xl` to make a list group horizontal starting at that breakpoint’s `min-width`.
Currently **horizontal list groups cannot be combined with flush list groups.**

<Example>
    <ListGroup
        horizontal
        list={[
            { content: 'WebCell' },
            { content: 'Cell Router' },
            { content: 'BootCell' }
        ]}
    />
</Example>

## Contextual classes

Use contextual classes to style list items with a stateful background and color.

<Example>
    <ListGroup
        list={[
            { content: 'WebCell' },
            { color: 'primary', content: 'A simple primary list group item' },
            {
                color: 'secondary',
                content: 'A simple secondary list group item'
            },
            { color: 'success', content: 'A simple success list group item' },
            { color: 'danger', content: 'A simple danger list group item' },
            { color: 'warning', content: 'A simple warning list group item' },
            { color: 'info', content: 'A simple info list group item' },
            { color: 'light', content: 'A simple light list group item' },
            { color: 'dark', content: 'A simple dark list group item' }
        ]}
    />
</Example>

```JavaScript
import { render } from 'web-cell';
import { ListGroup } from 'boot-cell/source/Content/ListGroup';

render(
    <ListGroup
        list={[
            { content: 'WebCell' },
            { color: 'primary', content: 'A simple primary list group item' },
            {
                color: 'secondary',
                content: 'A simple secondary list group item'
            },
            { color: 'success', content: 'A simple success list group item' },
            { color: 'danger', content: 'A simple danger list group item' },
            { color: 'warning', content: 'A simple warning list group item' },
            { color: 'info', content: 'A simple info list group item' },
            { color: 'light', content: 'A simple light list group item' },
            { color: 'dark', content: 'A simple dark list group item' }
        ]}
    />
);
```

> ##### Conveying meaning to assistive technologies
>
> Using color to add meaning only provides a visual indication,
> which will not be conveyed to users of assistive technologies – such as screen readers.
> Ensure that information denoted by the color is either obvious from the content itself (e.g. the visible text),
> or is included through alternative means, such as additional text hidden with the `.sr-only` class.
